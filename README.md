## 01단계

### 구현할 기능

1. 계좌개설
2. 입금
3. 출고
4. 전체고객 잔액조회

### 프로그램을 간결하게 하기 위한 몇 가지 가정

- 통장의 계좌번호는 중복되지 아니한다.(중복검사 하지 않겠다는 뜻)
- 입금 및 출금액은 무조건 0보다 크다.(입금 및 출금액의 오류검사 않겠다는 뜻)
- 고객의 계좌정보는 계좌번호, 고객이름, 고객의 잔액, 이렇게 세가지만 저장 및 관리한다.
- 둘 이상의 고객 정보 저장을 위해서 배열을 사용한다.
- 계좌번호는 정수의 형태이다.
